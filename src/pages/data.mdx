# Data

The DPI data is collected based on the [DPI conceptualisation and measurement](/methodology) framework. 

Each variable pertains to a specific attribute of DPI, tracking indicators across digital ID, digital payment and data exchange systems.

Read the [Codebook](/codebook) for detailed descriptions of the variables.

View the data in [Google Sheets][2025-10-27], or download the data as JSON:

<div class="grid grid-cols-1 sm:grid-cols-3 gap-2">
  <a href="/data/2025-10-27/2025-10-27-identity.json" class="button-blue" onclick="trackDownload('identity')"><IdentityIcon class='inline-block mr-2' /> Identity JSON </a>
  <a href="/data/2025-10-27/2025-10-27-payment.json" class="button-blue" onclick="trackDownload('payment')"><PayIcon class='inline-block mr-2' /> Payment JSON </a>
  <a href="/data/2025-10-27/2025-10-27-exchange.json" class="button-blue" onclick="trackDownload('exchange')"><DataIcon class='inline-block mr-2' /> Exchange JSON </a>
</div>

A snapshot of the data formatted for statistical analysis is available [here](https://drive.google.com/drive/folders/1dkIhzbEs6MQoULEjDhq_GXbuwMz-2Nxe)

<script>
{`
  // the wrapper is a workaround to allow us to put inline js inside mdx
  // see: https://github.com/withastro/astro/issues/6327#issuecomment-1449301347
  function trackDownload(label) {
    if (typeof gtag === 'function') {
      gtag('event', 'download', {
        'event_category': 'data_json',
        'event_label': label
      });
    } else {
      console.log(\`gtag not available. Analytics event not sent for: download, data_json, \${label}\`);
    }
  }
`}
</script>
